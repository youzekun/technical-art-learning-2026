# 教程与资料索引

本索引只服务于 **2026-07-18 至 2026-12-31 技术美术实习计划**。它不是“收藏夹”，而是各阶段的最小学习路径。

## 使用规则

1. 每个主题只保留一个主教程；遇到不懂的概念，再查一个参考资料。
2. 教程与动手时间至少保持 `1:1`。看完视频但没有代码、材质、工程、截图或说明，不计入完成。
3. 每学完一个小节，按“跟做一次 → 改参数观察 → 关掉教程重做 → 保存证据”验收。
4. UE 教程出现版本差异时，以当前 UE5 官方文档为准。
5. 材质阶段使用 UE5 传统材质系统。暂不同时学习 Substrate；旧 UE4 教程中的 Tessellation、World Displacement、旧版光追折射先跳过。
6. 英文视频不逐句翻译，只记录本节使用的节点、数学原理、参数和性能成本。

## 第 1—4 周：UE5、材质与 UE C++ 基础

### UE 编辑器

- 主教程：[UE5 First Hour](https://dev.epicgames.com/documentation/en-us/unreal-engine/first-hour-in-unreal-engine)

- C++ Actor、`.h`/`.cpp`、`BeginPlay`、`Tick`、`UPROPERTY`、`UFUNCTION`。

跳过完整游戏玩法、UI、角色控制器、联网和大型示例工程源码。

视频辅导只选一条主线：

- 一个支持 Base Color、Metallic、Roughness、Normal、UV 缩放的母材质。
- 金属、塑料、木材三个 Material Instance。
- 一张母材质节点图、一张实例参数截图、一段从空白重做的记录。

## 第 5—7 周：Shader 数学与效果初版

### 图形学定向复习

- 主资料：[GAMES101 课程主页](https://sites.cs.ucsb.edu/~lingqi/teaching/games101.html)
- 官方课程视频：[GAMES101 Bilibili](https://www.bilibili.com/video/av90798049/)

只按作品需要回看：

| 作品问题 | 回看内容 |
|---|---|
| 方向、法线、Fresnel | 向量、点积、归一化 |
| 世界空间积雪或苔藓 | 变换、坐标空间、法线 |
| WPO 顶点动画 | 模型变换、顶点处理 |
| PBR 材质 | 着色、纹理、材质与 BRDF 基础 |

辅助参考：

- [LearnOpenGL：Basic Lighting](https://learnopengl.com/Lighting/Basic-Lighting)
- [LearnOpenGL：Normal Mapping](https://learnopengl.com/Advanced-Lighting/Normal-Mapping)
- [LearnOpenGL：PBR Theory](https://learnopengl.com/PBR/Theory)
- [Freya Holmér](https://www.youtube.com/@acegikmo)：只查 `vectors`、`dot product`、`coordinate spaces`、`SDF`。

本阶段每个知识点必须在 UE 材质中做一个可视化验证。不要从头重刷 GAMES101。

### 效果实践

- [Tech Art Aid](https://www.youtube.com/@TechArtAid)：只按当前效果搜索 `dissolve`、`hologram`、`world aligned`、`vertex animation`。
- [Ben Cloward](https://www.youtube.com/@BenCloward)：用于补数学原理和材质组织方式。

最低产出：

- 溶解。
- 护盾或全息。
- 世界空间积雪或苔藓。
- 水面或 WPO 顶点动画。

截至 9 月 5 日，至少三个效果达到可展示状态，第四个完成核心逻辑，并形成第一批投递材料。

## 第 8—10 周：HLSL、后处理与作品一

按顺序学习：

1. [HLSL 语言参考](https://learn.microsoft.com/en-us/windows/win32/direct3dhlsl/dx-graphics-hlsl-reference)
2. [HLSL 内置函数](https://learn.microsoft.com/en-us/windows/win32/direct3dhlsl/dx-graphics-hlsl-functions)
3. [UE Custom Material Expressions](https://dev.epicgames.com/documentation/en-us/unreal-engine/custom-material-expressions-in-unreal-engine)
4. [UE Material Functions](https://dev.epicgames.com/documentation/en-us/unreal-engine/creating-and-using-material-functions-in-unreal-engine)
5. [UE Post Process Materials](https://dev.epicgames.com/documentation/unreal-engine/post-process-materials-in-unreal-engine?lang=en-US)

只学：

- `float`、`float2/3/4`、Swizzle、函数和简单条件。
- `dot`、`normalize`、`lerp`、`saturate`、`frac`、`step`、`smoothstep`。
- UV、遮罩、简单噪声或扭曲。
- Scene Depth、后处理材质和描边基础。

每个 Custom HLSL 效果要保留：

- 节点版与 HLSL 版的对照。
- 输入、输出、代码注释。
- 指令或复杂度截图。
- 为什么使用 Custom 节点，以及它可能带来的优化限制。

作品一的最终验收以[作品验收清单](./04_作品验收清单.md)为准。

## 第 11—15 周：UE 美术资产检查工具

主资料：

- [Scripting the Unreal Editor Using Python](https://dev.epicgames.com/documentation/unreal-engine/scripting-the-unreal-editor-using-python)
- [Editor Utility Widgets](https://dev.epicgames.com/documentation/en-us/unreal-engine/editor-utility-widgets-in-unreal-engine)

实现顺序：

1. 查询 Content Browser 中选中的资产。
2. 批量重命名。
3. 命名规范检查。
4. 贴图分辨率检查。
5. Static Mesh 材质槽数量检查。
6. 错误提示与 CSV/JSON 报告。
7. 安装说明、源代码和操作视频。

重要约束：移动或重命名 `.uasset` 时使用 Unreal 的资产 API，不使用系统文件操作直接改动 `.uasset`。

每增加一条规则，都准备“合格资产、违规资产、空选择或错误输入”三类测试。

## 第 16—19 周：性能分析与优化案例

按顺序学习：

1. [Performance Profiling and Configuration](https://dev.epicgames.com/documentation/en-us/unreal-engine/introduction-to-performance-profiling-and-configuration-in-unreal-engine)
2. [Unreal Insights](https://dev.epicgames.com/documentation/unreal-engine/unreal-insights-in-unreal-engine?lang=en-US)
3. [Render Resource Viewer](https://dev.epicgames.com/documentation/en-us/unreal-engine/render-resource-viewer-in-unreal-engine)
4. [Optimizing and Debugging for Realtime Rendering](https://dev.epicgames.com/documentation/en-us/unreal-engine/optimizing-and-debugging-projects-for-realtime-rendering-in-unreal-engine)
5. [Shader Complexity 与 Quad Overdraw 参考](https://dev.epicgames.com/documentation/unreal-engine/measuring-performance-in-niagara?lang=en-US)

固定测试协议：

- 同一台电脑。
- 同一分辨率与画质档位。
- 同一关卡、相机位置和运行模式。
- 预热后采样，记录 Game、Render、GPU 毫秒数。
- 同时记录 Shader Complexity、透明过绘、纹理、材质槽、Draw Call 和实例化情况。

不能用“直接关闭所有效果”作为主要优化。每项优化都要说明瓶颈、改动、数据变化和画质代价。

## 第 20—24 周：简历、投递与面试

这一阶段不再开启新的大课程，只围绕现有作品补缺口：

- 每个项目准备 30 秒、2 分钟和 5 分钟三种讲解。
- 每周至少 5 次有效投递或岗位触达，并记录岗位、日期、版本和反馈。
- 面试复习范围限定为 C++ 基础、向量与坐标空间、PBR/Shader、UE 工具、性能分析。
- 收到岗位反馈时，优先修正真实暴露的问题；没有反馈时，按作品验收清单补证据。
