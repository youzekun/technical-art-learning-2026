# 求职工作区

当前状态：**未开始制作正式材料**。

这里保存从9月开始投递所需的本地源文件和记录；公开版本最终发布到GitHub/Gitee与B站。

## 目录约定

```text
求职/
├─ README.md
├─ 模拟面试记录.md
├─ 简历/
│  ├─ 源文件
│  └─ PDF
└─ 视频/
   ├─ 项目单独视频
   └─ 作品总视频
```

简历和视频文件较大时可以只保留路径、版本和公开链接，不要把大型原始视频提交到Git仓库。

## 简历完成条件

- [ ] 一页中文简历。
- [ ] 教育经历、技能和项目时间线一致。
- [ ] 三个核心项目按“问题—行动—结果—个人职责”描述。
- [ ] 不写无法通过作品或问答证明的技能。
- [ ] 导出PDF并在另一台设备检查字体、链接和排版。
- [ ] 文件名包含姓名、方向和版本日期。

## 作品总视频完成条件

- [ ] 2—3分钟。
- [ ] 开头10秒展示方向和最佳成果。
- [ ] Shader、工具、性能案例都有明确段落。
- [ ] 画面中标注个人职责、关键技术和数据。
- [ ] 不使用无法说明来源或授权的素材。
- [ ] B站公开链接在未登录状态下可访问。

## 投递版本

每次投递都要在[`岗位投递表`](../记录/岗位投递.csv)填写：

- 简历版本。
- 作品集版本。
- 投递日期。
- 当前状态。
- 下次跟进日期。

修改简历或作品后使用新的版本号，不覆盖已经投递时使用的版本记录。

## 12月周指标

| 周期 | 有效投递或触达目标 | 实际数量 | 证据 |
|---|---:|---:|---|
| 12月1—6日 | ≥5 | | |
| 12月7—13日 | ≥5 | | |
| 12月14—20日 | ≥5 | | |
| 12月21—27日 | ≥5 | | |
| 12月28—31日 | ≥5 | | |

12月最后一周虽然只有4天，仍按用户设定的每周至少5次执行；12月五个记录周期合计至少25次有效投递或触达。
