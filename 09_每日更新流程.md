# GitHub每日更新流程

公开仓库：

> <https://github.com/youzekun/technical-art-learning-2026>

本仓库用于记录2026年技术美术学习、作品制作、投递和面试准备。每天只记录真实完成内容，不用空提交维持“连续打卡”。

## 每天需要提供什么

学习结束后，把以下内容发给Codex：

- 当天实际学习时间。
- 完成的课程或主题。
- 自己写的代码、UE工程路径或材质。
- 编译/运行结果。
- 截图、视频或性能数据。
- 能否不看教程重做。
- 遇到的问题与未完成任务。

没完成也如实汇报；未完成的必做项顺延，加分项删除，不要求第二天加倍。

## 仓库中的日期结构

```text
练习/YYYY-MM-DD/                  当天练习源文件
记录/每日汇报/YYYY-MM-DD.md       当天学习汇报与验收
记录/evidence/YYYY-MM-DD/         截图、运行结果和其他证据
记录/每日进度.csv                 总体时间与任务进度
```

作品进入正式制作后，工程说明和可公开源码放在：

```text
作品/01_Shader实验室/
作品/02_资产检查工具/
作品/03_性能优化案例/
```

## Codex每日处理步骤

1. 检查代码、工程、截图和解释是否互相对应。
2. 编译或运行能够在本机验证的代码。
3. 判断“完成、部分完成、未完成”和真正掌握程度。
4. 将原始证据整理到当天目录。
5. 更新日报、每日进度、阶段看板和证据矩阵。
6. 删除当天不再需要的加分任务，给出下一学习日任务。
7. 创建日期提交并推送到GitHub。

## 提交信息

正常学习日：

```text
daily: 2026-07-18 pointer basics and first material
```

只修正文档或工具：

```text
docs: update week 1 review
tool: improve environment check
```

一次提交可以包含多个相互关联的当天成果，但不能把不同日期伪装成同一天完成。

## 哪些内容不能提交

- UE的`Binaries`、`Intermediate`、`Saved`、`DerivedDataCache`。
- Visual Studio的`.vs`、临时数据库和编译产物。
- 访问令牌、密码、验证码和私钥。
- 含个人隐私的截图或文档。
- 未确认授权来源的大型美术素材。
- 没有必要展示的完整第三方工程。

大型UE资源需要先判断是否使用Git LFS；在没有确认前只提交源码、配置、说明和必要截图。

## 更新边界

Codex不会在没有收到学习结果时虚构每日进度，也无法在后台自动知道你当天做了什么。每次你提交日报、代码或截图后，才会执行验收、提交和推送。
