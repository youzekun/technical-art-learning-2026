# 作品验收清单

## 作品一：UE5程序化材质与Shader

### 必须内容

- [ ] 溶解与边缘发光。
- [ ] 护盾或全息效果。
- [ ] 世界空间积雪、苔藓或高度混合。
- [ ] WPO水面或顶点动画。
- [ ] 至少一个Custom HLSL效果。

### 工程质量

- [ ] 每个效果都有Material Instance和合理的参数默认值。
- [ ] 参数、材质、实例和Material Function命名清楚。
- [ ] 重复逻辑已整理为Material Function。
- [ ] 没有无用节点或无法解释的教程残留。

### 展示与讲解

- [ ] 1—2分钟最终视频。
- [ ] 最终效果、节点图和参数面板截图。
- [ ] 向量、点积、遮罩与坐标空间说明。
- [ ] Shader Complexity或相应性能截图。
- [ ] 使用限制、性能成本和个人职责说明。
- [ ] 能在5分钟内独立讲完需求、原理、实现、优化与结果。

## 作品二：UE5美术资产检查工具

### 固定接口

- 输入：Content Browser 当前选中的资产。
- 规则：命名、目录、贴图分辨率、Static Mesh 材质槽数量。
- 输出：界面结果列表及 CSV/JSON 报告。
- 失败策略：单项失败记录错误并继续处理剩余资产。

### 功能

- [ ] 批量重命名。
- [ ] 命名规范检查。
- [ ] 贴图分辨率检查。
- [ ] Static Mesh材质槽数量检查。
- [ ] 错误输入和异常提示。
- [ ] CSV或JSON报告输出。

### 测试

- [ ] 空选择时不会崩溃。
- [ ] 混合选择不同资产类型时能够正确处理。
- [ ] 合规资产不会被误报。
- [ ] 不合规资产会显示具体原因。
- [ ] 报告可以保存并重新打开。
- [ ] 重复执行不会产生错误状态。
- [ ] 单个失败不会中断整批任务。
- [ ] 使用至少 20 个测试资产。
- [ ] 使用专门准备的正确和错误资产完成回归测试。

### 交付

- [ ] 安装与使用说明。
- [ ] 源代码和操作视频。
- [ ] 说明工具解决的生产问题和节省的手工步骤。

## 性能优化案例

### 固定条件

- [ ] 记录CPU、GPU、内存、操作系统和UE版本。
- [ ] 固定分辨率、画质、关卡、相机和测试路径。
- [ ] 优化前后使用完全相同的测试条件。

### 数据与结论

- [ ] Game Thread、Render Thread和GPU耗时。
- [ ] Shader Complexity与透明过绘。
- [ ] 纹理、GPU资源、Draw Call、材质槽、LOD或实例化数据。
- [ ] 明确指出瓶颈及每项优化的原因。
- [ ] 展示优化前后数据与画面对比。
- [ ] 说明画质损失、适用范围或功能限制。
